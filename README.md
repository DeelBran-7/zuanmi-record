# 赚米记录

一个可以部署到 GitHub Pages 的静态 PWA，用来记录资产项目、年度目标、每笔投入/盈利/亏损，以及黄金浮动盈亏。

## 当前功能

- 稳定资产仪表盘首页
- 首次启动为空账本，不包含任何人的真实数据
- 可选载入演示模板查看效果
- 2025 / 2026 / 2027 年份切换
- 可新增 2028、2029 和之后年份；有对应年份记录时也会自动出现在年份切换里
- 2026 目标进度与年度回看
- 动态资产项目：新增、归档、恢复；归档只代表项目结束，仍计入目前资产
- 资产详情页：目前价值、本金、总营收、已花掉、留存收益、记录明细
- 记录类型：投入本金、回款、已实现盈利、已实现亏损、分红、花掉/消费、手续费、黄金买入、当前估值、备注
- 股票不自动算浮亏
- 黄金按克数、成本、当前金价计算浮动盈亏；支持手动填写或实时更新
- 本地保存、JSON 导入导出
- Supabase URL / anon key 配置位
- 邮箱密码登录、忘记密码重置邮件与云端整份账本同步

## 本地预览

```bash
python3 -m http.server 4173 --directory outputs/zuanmi-record
```

然后打开 `http://localhost:4173`。

## GitHub Pages 部署

1. 新建一个 GitHub 仓库。
2. 把 `outputs/zuanmi-record` 目录里的文件放到仓库根目录。
3. 在仓库设置里打开 Pages，选择从 `main` 分支部署。
4. 等 GitHub Pages 生成链接后，用 iPhone Safari 打开。
5. 在 Safari 分享菜单里选择“添加到主屏幕”，就能像 App 一样打开。

## 给别人使用

可以直接把部署后的链接发给别人。新用户第一次打开会看到初始化向导，默认是空账本，不会包含你的数据。

演示模板只是匿名假数据，用来展示页面效果；每个人的数据都保存在自己的浏览器本地。接入 Supabase 登录后，再按用户账号隔离云端数据。

## Supabase 同步

1. 创建 Supabase 项目。
2. 在 SQL Editor 运行 `supabase-schema.sql`。
3. 在 Authentication 里启用 Email provider。默认邮件服务适合小范围免费试用；公开使用或频繁重置密码建议配置自定义 SMTP。
4. 在 App 的“同步”页填写 Supabase URL 和 anon key。
5. 用邮箱和密码注册/登录；忘记密码时可以通过重置邮件设置新密码。
6. 点“上传本机账本”保存到云端；另一台设备登录同一个邮箱后点“拉取云端账本”。

当前同步采用 `app_states` 单表保存整份账本，便于个人使用和备份。表结构和 RLS 在 `supabase-schema.sql`。

Supabase 官方要求公开 schema 的表开启 RLS；策略里使用 `auth.uid() = user_id`，确保用户只能读写自己的账本。

## 实时金价

当前版本使用 `https://api.gold-api.com/price/XAU/CNY` 获取人民币计价的 XAU 每盎司价格，再除以 `31.1034768` 换算为人民币/克。

如果接口不可用，应用仍可手动填写当前金价，不影响记账和导入导出。

如果浏览器仍显示旧页面，先刷新一次；PWA 离线缓存更新后会优先拉取新文件。
